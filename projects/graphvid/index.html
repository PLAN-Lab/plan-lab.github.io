Why Graph-Based Video Control?

Image-to-video models must infer motion, object interactions, and physical evolution from a single static frame. This is inherently ambiguous: the same image can support many plausible futures depending on which objects should move, what relationships should change, and how entities should influence one another.

Existing controllable video generation methods often rely on dense trajectories, optical flow, segmentation masks, sparse boxes, or other geometric motion cues. While effective for specifying displacement, these controls can be difficult to author in scenes with multiple objects. Small errors in a few control signals can lead to globally implausible motion, including inconsistent contact, sliding, object penetration, desynchronized behavior, or failure to preserve the intended interaction.

GraphVid instead represents user intent as a directed relational interaction graph. This provides a higher-level semantic interface for single- and multi-object control while still grounding the requested dynamics in detected objects and spatial structure.

Trajectory control is low-level

Dense paths are hard to author for multi-object scenes, especially under occlusion, overlap, or contact.

Users think in interactions

Many desired motions are naturally relational: one object pushes, follows, lifts, supports, or moves relative to another.

Graphs preserve structure

Directed nodes and edges bind the requested dynamics to the correct entities and interactions.

Abstract

Controllable video generation remains challenging because precise multi-object interactions are difficult to specify using text prompts or low-level motion controls. Trajectory-based interfaces often require users to draw accurate tracks for multiple objects, which becomes cumbersome as scene complexity increases and can become ambiguous under occlusion or overlap.

We introduce GraphVid, a graph-conditioned image-to-video generation framework that enables interactive control through structured interaction graphs. Starting from a single input frame, GraphVid constructs a directed scene graph over detected entities, allows users to modify object-level interaction edges, and translates the edited graph into conditioning tokens for a frozen video diffusion transformer.

To support this control paradigm, we curate GraphVid-Bench, an interaction-centric dataset of approximately 27K video clips paired with structured relational annotations. Despite using substantially fewer trainable parameters and less training data than existing motion-control methods, GraphVid achieves competitive perceptual quality, reconstruction fidelity, and inference efficiency.

✅ Contributions

  • Interactive graph-based control. GraphVid turns controllable image-to-video generation into an interactive graph editing problem, replacing dense trajectory authoring with object-level interaction edits.

  • Structured interaction representation. Nodes correspond to detected scene entities, while directed edges encode semantic dynamics such as push, pull, lift, hold, support, contact, move, and rotate.

  • Edge-aware graph reasoning. GraphVid injects edge attributes directly into graph message passing, allowing object representations to reflect both neighboring entities and the semantics governing their interactions.

  • Parameter-efficient graph-to-video adaptation. The video diffusion transformer and VAE remain frozen, while only the Edge-Aware GNN, graph-to-video adapter, and LoRA modules are trained.

  • GraphVid-Bench. We curate approximately 27K interaction-centric clips with structured graph annotations.

GraphVid Method

GraphVid is a controllable image-to-video framework that interfaces a structured interaction graph with a large pretrained video diffusion transformer. Given a conditioning image, GraphVid constructs an attributed directed graph where each node represents a detected entity and each directed edge describes how one entity influences another. Unary actions such as rotate, move, or scale are represented as self-loop edges.

Loading...

GraphVid overview. Objects are detected from the input image and encoded into node embeddings. An MLLM extracts relational edges to construct an initial interaction graph. Users edit interactions through the image interface, and the resulting graph is converted into conditioning tokens that guide the frozen video diffusion transformer.

Node representations. For each detected entity, GraphVid fuses a visual embedding from the object crop, a text embedding of the object label, and normalized bounding-box coordinates. This representation captures appearance, semantic identity, and spatial grounding.

Edge representations. Each directed edge is represented with an open-vocabulary textual descriptor of the intended interaction between source and target entities. These edge descriptors are embedded into dense edge features, allowing the model to distinguish different physical and semantic relationships rather than treating all graph connections as identical.

Edge-aware graph reasoning. Standard graph convolutional networks typically treat edges as binary connectivity. GraphVid instead uses an edge-aware graph reasoning module that injects edge attributes directly into message passing.

Edge-Aware Graph Reasoning. Node and edge embeddings are projected into a shared hidden space and processed by an edge-aware GCN. The resulting interaction-aware node embeddings are mapped into the hidden space of the video transformer and used as graph conditioning tokens.

GraphVid-Bench Dataset

Current video generation benchmarks often rely on text prompts or low-level motion cues, which provide limited control over object-level interactions. GraphVid-Bench is designed to support structured relational control by pairing videos with directed interaction graphs. Nodes represent scene entities, while edges encode the type and direction of interaction cues that influence scene dynamics.

GraphVid-Bench contains approximately 27K interaction-centric video clips. Each graph includes visual, semantic, and spatial node attributes, while edges capture interaction semantics and directionality.

Quantitative Results

GraphVid is evaluated on an interaction-centric MoveBench subset against trajectory-based, physics-aware, and motion-control video generation baselines. Wan-Move represents the strongest high-capacity baseline in this comparison, while GraphVid focuses on efficient structured control: it uses only 27K training clips and 0.3B trainable parameters while remaining competitive on perceptual quality and reconstruction fidelity.

Reading the table: the blue-highlighted Wan-Move row indicates the strongest high-capacity/SOTA baseline, while the orange-highlighted GraphVid row indicates our efficient graph-conditioned model. GraphVid is not positioned as simply replacing a 14.5B-parameter system; instead, it shows that structured interaction graphs can provide strong control and competitive quality with far less trainable capacity and data.

Qualitative Results

GraphVid generates interaction-consistent videos from a single image and user-specified graph edits. Across object-object, human-object, and multi-agent scenes, the model preserves scene identity while producing temporally coherent motion that follows the requested relational dynamics.

Interaction-conditioned generation. Given an input image and interaction graph, GraphVid generates coherent videos for diverse dynamics, including animals moving relative to one another, human-object interactions, human motion, and object manipulation.

Compared with prior controllable video generation systems, GraphVid better preserves the relationship between interacting entities. In multi-object scenarios, baseline methods often fail to separate object trajectories, introduce artifacts, generate the wrong action, or lose subject identity. GraphVid uses directed graph conditioning to bind the requested interaction to the correct entities, improving temporal consistency and structural reliability.
